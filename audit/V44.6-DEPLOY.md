## Rollback Plan

### Trigger conditions for rollback (any one):
- Live PF over first 30 days < 1.20
- Live DD > 5%
- Critical bug (race conditions, wrong sizing, signal duplicates)
- Reconciliation divergences vs Binance positions

### Rollback procedure (~60 seconds)
1. Render dashboard → service → **Environment** tab.
2. Set `APEX_V46_T6=0` and `APEX_V46_T5=0` (or delete entirely).
3. Click **Save Changes** → triggers redeploy (~2 min).
4. Verify boot logs show:
   ```
   [SignalGen] started — scan every 60000 ms, engine: apex-v44-funding-carry-1.0
   ```
5. V44.5 baseline restored. No frontend changes needed (stats display is informational, not causal).

### Why rollback is safe
- T6 + T5 are **multiplicative** size adjustments. Setting flags OFF makes both functions return 1.0 (no-op).
- No DB schema changes — `engine_version` field accepts any string.
- Existing in-flight trades are managed by their own engine version stored at creation; they close per their original logic.
- New trades after rollback use V44.5 baseline.
